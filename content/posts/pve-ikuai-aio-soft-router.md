手头有台老的 i7-7500U 小主机，16G 内存 512G 固态，正好两条宽带闲着没整合，就想着搭一套 AIO 软路由：PVE 9.2 当底层，上面跑 iKuai 当主路由，把电信（1000 下 50 上）和广电（100 对称）两条线做成负载均衡。这台机器有三个网口，板载两个加一块 miniPCIe 转出来的一个，刚好够用。

## 先把架构想清楚

我一开始的毛病就是上来插线开干，结果装到一半发现网段全错、后台进不去，白折腾。所以建议动手前先把链路在脑子里过一遍。我要的最终形态是：电信光猫和广电猫各出一根线进 iKuai 的两个 WAN 口，iKuai 做负载均衡，再从 LAN 口出来接全屋。三个网口的分工就是板载口一接电信当 WAN1，扩展口接广电当 WAN2，板载口二当 LAN。双宽带想两条线一起用，至少得 2 个 WAN 加 1 个 LAN，也就是最少三个口，所以扩展卡得在装系统前就装好，不然后面架构要推倒重来。我就是因为一开始以为扩展卡丢了、按双口配，后来翻出来又得重弄，还是得一步到位。

还有个前提得先确认：光猫是路由模式还是桥接模式。判断很简单，设备直插光猫不用填账号密码就能上网，就是路由模式，iKuai 的 WAN 用 DHCP 自动获取就行，这种是双 NAT。要是得填宽带账号密码拨号，那是桥接，iKuai 得自己配 PPPoE。我家两条都是路由模式，所以全程 DHCP。

## 装 PVE

直接使用 U 盘安装即可，我用的是 Ventoy，挺方便的。

网络配置这步是新手最容易卡的。Hostname 随便填个像样的比如 pve.lan，别留默认那个 pve.example.invalid。IP 这里要填一个跟光猫同网段、且没被占用的地址，我的光猫是 192.168.1.1 我就填 192.168.1.100，CIDR 填 24，网关填光猫的 192.168.1.1，DNS 填 192.168.1.1 或者 223.5.5.5。这里千万别用 DHCP，管理地址必须固定，不然哪天 IP 变了你就找不到后台了。我一开始想填 192.168.1.2，结果那地址被现有路由器占了，换成靠后的 192.168.1.100 就没事，这种靠后的地址基本不会撞。顺带说一句，装系统这步根本不用联网，网络配置只是填表，没插网线照样填完往下装，IP 写死之后搬到光猫旁边插线就生效，不用为了装系统把机器和显示器搬来搬去。文件系统单机用 ext4 就行，时区 Asia/Shanghai，管理员邮箱填个格式合法的比如 admin@pve.lan，安装器会校验格式，纯写 admin 会被打回，但这玩意儿家用根本不影响使用。

装完自动重启，拔 U 盘，浏览器开 `https://你的IP:8006` 进后台，root 加你设的密码。

## 进后台先换源

PVE 默认境外源，国内慢到下不动东西，第一件事换国内镜像。节点 → Shell 打开命令行，先看版本，PVE 9.2 是基于 Debian 13（trixie）的：

```bash
cat /etc/debian_version
```

Debian 13.x 就用下面这套，要是 12.x 就把 trixie 全换 bookworm：

```bash
sed -i 's|http://deb.debian.org|https://mirrors.tuna.tsinghua.edu.cn|g' /etc/apt/sources.list.d/debian.sources

echo 'Types: deb
URIs: https://mirrors.tuna.tsinghua.edu.cn/proxmox/debian/pve
Suites: trixie
Components: pve-no-subscription
Signed-By: /usr/share/keyrings/proxmox-archive-keyring.gpg' > /etc/apt/sources.list.d/pve-no-subscription.sources

apt update
```

企业源会一直报 401，强迫症的话关掉。注意 PVE 9 用的是新的 deb822 格式，企业源报错是因为 URL 指向 enterprise.proxmox.com，光改 Components 字段没用，直接加 Enabled false 最干脆：

```bash
echo 'Enabled: false' >> /etc/apt/sources.list.d/ceph.sources
echo 'Enabled: false' >> /etc/apt/sources.list.d/pve-enterprise.sources
apt update
```

ceph 源软路由用不上，那是分布式存储的东西，关掉无所谓。

## 建 iKuai 虚拟机

iKuai 官网下 x64 的 ISO，PVE 后台 local 存储那里 Upload 传上去。然后创建虚拟机，几个要点：类型选 Linux 6.x，机型选 q35（想留 PCIe 直通的后路就这个），BIOS 用 SeaBIOS。这个挺关键，iKuai 是传统 BIOS 引导，选了 UEFI 装完会黑屏起不来。注意别跟 iStoreOS 搞混，iStoreOS 的 efi 版才用 UEFI，两个引导方式不一样。磁盘给 8 到 32G 都行，记得勾上 Discard，也就是 SSD 的 TRIM，配合精简置备能做到给 32G 但实际不占真实空间，不勾的话会实打实占满。CPU 类型选 host 把指令集透传给虚拟机性能最好，核心给 2 个就够。

i7-7500U 总共 4 个逻辑核，iKuai 给 2、留 2 个给旁路由和 PVE，正好按需抢核。给 iKuai 4 核会让它忙起来把整颗 CPU 包圆，旁路由抢不到，全家网跟着卡。软路由是转发型的，看的是多包并发不是单核性能，2 核跑满千兆毫无压力。内存 2048，网络挂 vmbr0、型号 VirtIO，防火墙那个勾一定取消掉，PVE 的防火墙会拦软路由流量，NAT 转发各种不通。

建完去虚拟机的 Options 里把 Start at boot 改成 Yes，开机自启，不然断电恢复后全家没网还得手动来开。

## 给 PVE 配多网桥

iKuai 要管三个口，做法是给每个物理口建一个独立网桥，再挂给 iKuai 当三块网卡。先去节点 → 系统 → 网络，看清三个口的名字和 MAC。我这里是 nic0（板载，接光猫，装机时已经在 vmbr0 里）、nic1（板载，空）、enp3s0（扩展卡，空）。板载双口的 MAC 一般连号，扩展卡的 MAC 段完全不同，靠这个区分。

建两个新桥，创建 → Linux Bridge，IP 都留空：vmbr1 桥接 enp3s0 给广电用，vmbr2 桥接 nic1 给 LAN 用。建完一定要点"应用配置"让它生效，光建出来只是草稿。我就是没点，直接给 iKuai 加网卡，报了个 bridge vmbr1 does not exist。然后给 iKuai 加两块网卡，硬件 → 添加 → 网络设备，分别桥 vmbr1 和 vmbr2，型号 VirtIO，防火墙取消勾选。这样 iKuai 就有三块网卡，net0 是 vmbr0 接电信、net1 是 vmbr1 接广电、net2 是 vmbr2 当 LAN，iKuai 内部认成 eth0/eth1/eth2。

## 装 iKuai

开机进控制台，输 1 把系统装到硬盘，装完输 R 重启。装完记得在 PVE 硬件页把那个 ISO 光驱设成不使用介质或者移除，引导顺序里硬盘放第一，不然重启又进安装界面死循环。

重启后 iKuai 默认管理地址是 192.168.9.1。先在控制台里把 LAN 网卡绑到 eth2：

```
del lan1
set lan1 eth2
```

这样 eth2 就是 LAN，eth0/eth1 留给 WAN。

## 配 WAN 让网通起来

iKuai 的 LAN 是 192.168.9.1，跟光猫网段不一样，得让笔记本进 192.168.9.x 这个网。把笔记本网线插到 PVE 的 nic1（LAN 口），DHCP 还没配好拿不到地址的话，先给笔记本手动配个静态：IP 192.168.9.100、掩码 255.255.255.0、网关留空、DNS 留空。网关留空这点很重要，配了网关 Windows 可能把默认路由抢去走网线，而这会儿 iKuai 还没通外网，结果笔记本断网。留空的话访问 192.168.9.x 走网线进后台，其它外网流量走 Wi-Fi 保持能上网，Windows 同时连网线和 Wi-Fi 互不影响。另外如果你开着 Mihomo / Clash 之类的代理，可能进不去 192.168.9.1。配置正经的代理默认会绕过 192.168 私有网段，一般不影响，真进不去就关掉代理或者在规则里加 192.168.9.0/24 直连。

浏览器开 192.168.9.1，默认 admin / admin，首次登录强制改密码。

进去后网络设置 → 内外网设置，编辑 wan1，网卡用途选 WAN，选 eth0 点绑定，接入方式选 DHCP，保存。几秒钟 wan1 就从光猫拿到地址了，状态显示已连接外网就通了。绑定网卡时 iKuai 会重载网络，可能把你登出，正常现象重新登就行。广电同理，把广电猫的线插到 enp3s0，新建 wan2 绑 eth1，接入方式 DHCP，保存。广电猫也是路由模式自动发地址，状态已连接就成了。

## 配双线负载均衡

两条 WAN 都通了，流控分流 → 分流设置 → 多线负载，点添加。负载模式选「源IP+目的IP」，同一设备访问同一网站固定走一条线，最稳，免得敏感网站因为 IP 跳变要你重新验证。运营商选全部。负载比例 wan1 wan2 都点启用，设权重，我按带宽差设了 8 比 1，电信带宽是广电十倍。保存就生效了。

顺便说个很多人担心的：两个 IP 请求同一内容会不会被平台封号。不会。负载均衡是按会话分线不是把一个请求拆两条线发，你打开一个网页或下一个文件，这个会话从头到尾走同一条线，平台看到的就是一个正常 IP 的完整请求，跟手机从 Wi-Fi 切 5G 一个道理。这是千万家庭和企业的标准用法。再说了，就算是两个 IP，但都是同一个地区的，根本不需要担心什么，现代风控系统没有那样严格，否则用移动数据的用户都遭殃了。

还有一点别误会，下行不会简单 1000 加 100 等于 1100，负载均衡是多会话分摊，单个下载只走一条线最高还是 1000。但多任务多设备同时跑两条线一起上，总吞吐能超 1000。广电那 100 对称上行很适合留给上传场景。

## 配 LAN 的 DHCP

这里踩了个大坑。配完所有东西，笔记本改成自动获取 IP 却一直拿不到地址，老是 169.254 那个兜底地址。去网络设置 → DHCP设置 → DHCP服务端一看，服务端状态是「服务已关闭」，lan1 那条规则状态「已失效」，原来 DHCP 服务压根没开。池配置本身没问题，地址池、网关、DNS 都对，就是服务没启用发不出地址。勾上 lan1 那行点启用，再点那个红色的重启 DHCP 服务就好了。修好后笔记本以太网改回自动获取，命令行 ipconfig /release 再 ipconfig /renew，就能拿到 192.168.9.x 正常上网了。验证就是把 Wi-Fi 禁用只留网线，能开网页就说明 iKuai 主路由彻底通了。

说个排查技巧，网页打不开别急着怪软路由。ping 223.5.5.5 直接 ping IP 不走 DNS，通了说明 iKuai 转发没问题，问题在浏览器或代理层；不通才是 iKuai 的事。再 nslookup qq.com 223.5.5.5 看解析，要是解析出 198.18 开头的地址，那是代理软件的 fake-ip 在劫持 DNS，跟软路由无关。我当时就是这么分清楚的，省了一堆时间。

## 到这就通了

PVE 装好换了源，iKuai 双 WAN 负载均衡 8 比 1，LAN DHCP 正常设备自动拿地址能上网，开机自启也设了。一套双线 AIO 软路由的核心就跑起来了。

## 总结

旁路由这块还没上，打算装个 iStoreOS 跑广告过滤、Docker、网络存储那些插件，得把 img.gz 传进 PVE 写盘建虚拟机再设旁路由网关，下次再搞。

物理这块也得收尾。机器现在还是裸主板放外面，得装进机箱，裸板带电被宠物碰可能静电或短路烧硬件，软路由是 7×24 跑的全家网络命脉，必须封起来。另外想把闲置的旧路由器刷成 AP 模式纯发 Wi-Fi，关掉它的路由和 DHCP 用 LAN 口接 iKuai。家里要是有人不想被折腾影响，可以单独留一台路由器独立挂光猫给他用，互不干扰。

可选的优化还有策略分流，下载强制走电信、上传走广电，以及把第三个口接下游交换机或 AP。这些以后慢慢弄。

最后，i7-7500U 这种老 U 跑双线千兆完全够，闲置时 CPU 才几个百分点，别被移动 U 性能弱吓住，转发看的是并发不是单核。
